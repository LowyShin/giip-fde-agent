# Human-in-the-loop 승인 게이트 워크플로우

이 워크플로우는 다음 작업이 실행되기 전에 반드시 사람 승인을 받도록 강제합니다.

## 1. 승인 게이트 대상

아래 항목 중 하나라도 해당하면 승인 게이트를 통과해야 합니다.

- 배포(Production 반영, 릴리스 태깅, 인프라 변경 포함)
- 비용 증가(유료 API/토큰/서버 비용이 증가하는 설정 변경)
  - 기준: 월 예상 추가비용이 **$100 이상**이거나, 기존 대비 **20% 이상 증가** 중 **하나라도 충족**하면 필수 승인
- 외부 연동(신규 SaaS/API/Webhook/메신저 연동)
- 민감 데이터 접근(개인정보/인증정보/내부 기밀 데이터 처리)

## 2. 게이트 단계

1. **사전 요약 작성**
   - 목적
   - 영향 범위
   - 롤백 방법
   - 예상 비용 변화
2. **검토자 지정**
   - 최소 1명 이상의 책임자(Owner/Reviewer)
3. **승인 기록**
   - 승인자 이름, 시각, 승인 코멘트
4. **실행**
   - 승인 이후에만 실행
5. **사후 검증**
   - KPI 영향 및 장애 여부 확인

## 3. 실패/거절 시 처리

- 승인 거절 시 작업 상태를 `Pending`으로 되돌리고 수정 계획을 첨부합니다.
- 승인 없이 실행된 경우 `Incident`로 기록하고 즉시 롤백 절차를 시작합니다.

## 4. 최소 체크리스트

- [ ] 고위험 여부 판정 완료
- [ ] 사전 요약 작성 완료
- [ ] 검토자 지정 완료
- [ ] 승인 기록 완료
- [ ] 실행 후 검증 완료
