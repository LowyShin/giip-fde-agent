# Tool Usage & Verification

도구 사용과 컨텍스트 검증에 관한 원칙. 정보의 신뢰성을 높이고 불필요한 도구 호출을 줄인다.

## 1. 파일 존재/컨텍스트 검증

**암시된 파일도 실제 존재 여부를 확인하고 사용한다.**

- 사용자가 파일이 있다고 말해도, 실제로 없을 수 있다. 읽기/편집 전에 반드시 확인.
- 디렉터리 구조나 경로를 추정하지 않는다. `glob` 또는 `view`로 직접 확인.
- 오래된 컨텍스트(이전 대화, 캐시된 정보)는 현재 repo 상태와 다를 수 있다.

## 2. 최신 정보 검증

**정보의 변동성에 따라 검증 방법을 선택한다.**

| 정보 유형 | 처리 방법 |
|-----------|-----------|
| 언어 문법, 알고리즘, 수학 | 정적 지식으로 즉시 답변 |
| 라이브러리 API, 버전 | 최근 변경 가능성이 있으면 공식 문서 확인 |
| 현재 직책, 정책, 가격, 최신 이벤트 | 반드시 검색 후 답변 |
| repo 내 코드/설정 | repo 파일을 직접 읽어서 확인 |

- 확인 없이 추정한 최신 정보를 사실처럼 제시하지 않는다.
- 불확실하면 "최신 정보는 직접 확인하세요"라고 안내한다.

## 3. 도구 선택 원칙

**query 성격에 맞게 최소한의 도구를 사용한다.**

우선순위:
1. **로컬 파일 확인 우선** — repo 내 파일로 답할 수 있으면 먼저 읽는다.
2. **repo 내 evidence 우선** — 코드, 설정, 문서에서 근거를 찾는다.
3. **공식 문서 우선** — 외부 정보가 필요하면 공식 출처를 참조한다.
4. **최신성 필요 시 웹 검증** — 변동성 높은 정보만 검색한다.

- 검색은 정적 지식으로 답할 수 없을 때만 실행한다.
- 동일한 정보를 얻기 위해 여러 도구를 중복 호출하지 않는다.
- 작업 전 `.agent/skills/`의 관련 SKILL.md를 먼저 읽는다.

## 4. 공식 문서/원문 우선

**공식 출처와 repo 내 실제 코드를 최우선 근거로 삼는다.**

- 블로그, 스택오버플로 등 비공식 출처는 공식 문서와 충돌 시 공식 문서를 따른다.
- repo 내 코드가 문서와 다르면 코드를 사실로, 문서를 갱신 대상으로 본다.
- 외부 자료를 인용할 때는 출처 URL을 명시한다.
