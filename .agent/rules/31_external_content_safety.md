# External Content Safety

외부 자료 처리 시 저작권과 라이선스 리스크를 최소화하는 원칙.

## 1. 외부 자료 직접 인용 최소화

**원문을 그대로 붙여넣지 않는다.**

- 외부 문서, 기사, 코드의 긴 원문을 그대로 재현하지 않는다.
- 핵심 개념만 추출하여 자기 말로 재서술한다.
- 인용이 필요하면 출처 URL과 함께 짧은 발췌(1~2문장)만 사용한다.

## 2. 긴 문단 재현 금지

**요약은 재서술(paraphrase) 중심으로 작성한다.**

- 책, 블로그, 공식 문서의 단락을 그대로 복사하지 않는다.
- 여러 출처를 종합할 때는 원문 구조를 그대로 따르지 않는다.
- 번역도 직역보다 의역(내용 중심 재서술)을 우선한다.

## 3. 저작권/라이선스 민감성 주의

**라이선스를 확인하지 않은 외부 코드를 repo에 포함시키지 않는다.**

- 외부 코드 snippet을 그대로 복사하기 전에 라이선스를 확인한다.
- MIT, Apache 2.0 등 허용 라이선스 외의 코드는 포함하지 않는다.
- GPL 계열 코드는 repo 전체 라이선스에 영향을 줄 수 있으므로 주의한다.
- 라이선스가 불명확한 외부 코드는 직접 포함하지 않고 링크로 대체한다.

## 4. README/가이드 갱신 시 적용

**문서화 작업에서도 위 원칙을 따른다.**

- 외부 가이드를 참고해 README를 작성할 때 내용을 재서술한다.
- 직접 인용이 필요하면 출처를 명확히 표기하고 최소 분량으로 제한한다.
- 자동 생성 문서에도 외부 원문이 그대로 포함되지 않도록 검토한다.
