# AI Native 운영 KPI 표준

이 문서는 GIIP Agent System의 운영 성과를 주간 단위로 측정하기 위한 최소 KPI 표준을 정의합니다.

## 1) 핵심 KPI

| KPI | 정의 | 계산식 | 목표(초기 권장) |
|---|---|---|---|
| 리드타임(Lead Time) | 요청 접수부터 완료 보고까지 걸린 시간 | 완료시각 - 요청시각 | 기준선 대비 개선 후 허용 범위 내 안정화 |
| 자동화 성공률 | 사람 개입 없이 완료된 작업 비율 | 자동 완료 건수 / 전체 작업 건수 | 70% 이상 |
| 재작업률 | 완료 후 재오픈/재수정된 작업 비율 | 재작업 건수 / 완료 건수 | 15% 이하 |
| 인간개입률(HITL) | 승인/검토/수동수정으로 인간이 개입한 비율 | 인간개입 건수 / 전체 작업 건수 | 위험 구간만 선택 개입 |
| 장애 건수 | 운영 중단/심각 오류 발생 건수 | 주간 Critical+High incident 수 | 0 목표 |

## 2) 수집 원칙

- 모든 작업은 최소한 다음 메타데이터를 남깁니다: `시작시각`, `완료시각`, `자동완료 여부`, `재작업 여부`, `인간개입 여부`, `장애 연관 여부`.
- 샘플 기반이 아닌 전체 작업 집계를 기본으로 합니다.
- 숫자만 기록하지 말고, KPI 변동 원인과 다음 액션을 함께 기록합니다.

## 3) 해석 가이드

- 자동화 성공률이 높아도 장애 건수/재작업률이 오르면 품질 저하로 판단합니다.
- 인간개입률은 낮을수록 좋은 지표가 아니라, 위험도 기반으로 적절히 유지되어야 합니다.
- 리드타임 개선은 승인 게이트/검증 단계 생략이 아닌 병목 제거로 달성해야 합니다.

## 4) 운영 루프

1. 주간 KPI 리포트 작성
2. 원인 분석(병목/환각/외부 API 이슈)
3. 다음 주 실험 항목 정의(최대 3개)
4. 워크플로우/스킬/룰 반영
5. 다음 주 동일 KPI로 효과 검증
