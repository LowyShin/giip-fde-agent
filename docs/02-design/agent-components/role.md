# 에이전트 구성 요소: 역할 (Roles)

역할(Role)은 AI 에이전트가 특정 작업을 수행할 때 채택하는 **페르소나와 책임 범위**를 정의합니다. 각 역할은 고유한 전문 지식과 행동 지침을 가지고 있습니다.

## 📝 역할의 정의
역할은 에이전트가 "누구로서" 작업을 수행할지를 결정합니다. 예를 들어, `developer` 역할은 코드 구현에 집중하고, `analyst` 역할은 문제의 근본 원인을 파악하는 데 집중합니다.

## 🛠️ 작성 및 유지 관리 방법
*   **저장 위치**: `.agent/roles/` 디렉토리에 마크다운(`.md`) 파일로 저장됩니다.
*   **구성 요소**:
    *   **Persona**: 에이전트의 성격과 전문 분야.
    *   **Mission**: 해당 역할이 달성해야 하는 궁극적인 목표.
    *   **Core Values**: 작업 시 우선순위를 두어야 할 가치 (예: 안전, 효율, 가독성).
    *   **Response Guidelines**: 사용자에게 응답하는 방식과 톤앤매너.
*   **수정 방법**: 기존 `.md` 파일을 수정하거나 새로운 파일을 추가하여 시스템에 즉시 반영할 수 있습니다.

## 👤 유저가 해야 할 부분
*   **역할 선택**: 작업의 성격에 맞는 역할을 명시하거나, 시스템이 자동으로 선택하도록 유도합니다. (예: "개발자로서 이 기능을 구현해줘")
*   **역할 정의**: 프로젝트에 특화된 새로운 전문가가 필요할 경우, 요구사항을 바탕으로 새로운 역할 파일을 생성하도록 AI에게 요청할 수 있습니다.

## 🤖 AI가 자동으로 해주는 부분
*   **문맥 전환**: 선택된 역할에 맞춰 사고 방식, 도구 사용 전략, 응답 스타일을 즉시 변경합니다.
*   **도구 최적화**: 역할에 최적화된 도구(Skills)를 우선적으로 탐색하고 활용합니다.
*   **자동 할당**: `launch_subsession.ps1`과 같은 스크립트 실행 시, 작업 내용을 분석하여 가장 적합한 역할을 자동으로 선택하여 실행합니다.
