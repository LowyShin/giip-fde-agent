# 에이전트 구성 요소: 워크플로우 (Workflows)

워크플로우(Workflow)는 여러 단계로 이루어진 **복잡한 작업의 실행 절차**를 정의합니다. 슬래시 명령어(`/`)를 통해 에이전트에게 정해진 프로토콜대로 작업을 수행하도록 명령할 수 있습니다.

## 📝 워크플로우의 정의
워크플로우는 "작업의 순서도"입니다. 단순히 한 번의 명령으로 끝나는 것이 아니라, 계획-실행-검토-보고와 같은 일련의 과정을 에이전트가 스스로 완수하게 합니다.

## 🛠️ 작성 및 유지 관리 방법
*   **저장 위치**: `.agent/workflows/` 디렉토리에 마크다운(`.md`) 파일로 저장됩니다.
*   **연동 방식**: 파일 이름이 곧 슬래시 명령어가 됩니다 (예: `native-trace.md` -> `/native-trace`).
*   **작성 규칙**: 각 단계(Step)를 명확하게 구분하고, 에이전트가 해당 단계에서 수행해야 할 도구 호출이나 사고 과정을 기술합니다.

## 👤 유저가 해야 할 부분
*   **명령 실행**: 필요한 시점에 `/plan`, `/native-trace` 등의 명령어를 입력하여 정해진 절차를 시작합니다.
*   **절차 정의**: 팀 내에서 반복적으로 발생하는 업무 프로세스를 워크플로우로 만들어 공유합니다.

## 🤖 AI가 자동으로 해주는 부분
*   **단계별 실행**: 정의된 워크플로우의 순서에 따라 다음 단계를 자동으로 판단하고 수행합니다.
*   **상태 추적**: 워크플로우 진행 상황을 기록하고, 중간에 오류가 발생하더라도 어디서부터 다시 시작해야 하는지 파악합니다.

---

## 🚀 초보자를 위한 커스텀 워크플로우 만들기

나만의 워크플로우를 만드는 것은 매우 간단합니다. 다음 단계를 따라해보세요.

### 1단계: 워크플로우 파일 생성
`.agent/workflows/` 폴더에 원하는 이름으로 마크다운 파일을 만듭니다.
> 예: `my-update.md` (이 파일은 `/my-update` 명령어가 됩니다)

### 2단계: 설명 추가 (Frontmatter)
파일의 맨 위에 이 워크플로우가 무엇을 하는지 적어줍니다. 에이전트가 이 내용을 보고 언제 사용할지 판단합니다.
```markdown
---
description: 프로젝트의 특정 부분을 자동으로 업데이트하고 보고하는 워크플로우입니다.
---
```

### 3단계: 단계별 지침 작성
에이전트가 수행해야 할 일을 번호를 매겨 상세히 적습니다.
```markdown
# /my-update 워크플로우

## 1. 현황 파악
먼저 `src/` 폴더의 내용을 리스트업하여 변경 사항이 있는지 확인합니다.

## 2. 코드 수정
확인된 내용을 바탕으로 필요한 코드를 수정합니다. 이때 `developer` 역할을 참고하세요.

## 3. 테스트 및 보고
수정이 끝나면 `npm test`를 실행하고 결과를 요약하여 보고서로 작성합니다.
```

### 4단계: 사용하기
이제 채팅창에 `/my-update`라고 입력해보세요. 에이전트가 당신이 정의한 1, 2, 3단계를 차례대로 수행하기 시작합니다.

> [!TIP]
> **자동 실행 팁**: 단계 설명 위에 `// turbo` 주석을 달아두면, 에이전트가 유저의 확인을 기다리지 않고 해당 단계의 명령어를 자동으로 실행합니다.
