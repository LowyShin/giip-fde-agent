# GIIP 에이전트 시스템 구성 요소 (Agent Components)

GIIP 에이전트 시스템은 유기적으로 연결된 네 가지 핵심 구성 요소로 이루어져 있습니다. 각 구성 요소의 상세 내용은 아래 링크를 통해 확인하실 수 있습니다.

## 🧱 핵심 구성 요소

### 1. [역할 (Roles)](./role.md)
*   **정의**: AI가 작업 시 채택하는 페르소나와 책임 범위.
*   **핵심**: "누구로서" 일할 것인가?
*   **위치**: `.agent/roles/`

### 2. [규칙 (Rules)](./rule.md)
*   **정의**: 모든 작업 시 반드시 준수해야 하는 강제 지침.
*   **핵심**: "어떤 제약"을 지킬 것인가?
*   **위치**: `GEMINI.md`, `.agent/rules/`

### 3. [스킬 (Skills)](./skill.md)
*   **정의**: 특정 도구나 기술을 다루기 위한 전문 지식 패키지.
*   **핵심**: "무엇을 할 수" 있는가?
*   **위치**: `.agent/skills/`

### 4. [워크플로우 (Workflows)](./workflow.md)
*   **정의**: 복잡한 작업의 실행 절차 및 슬래시 명령어 프로토콜.
*   **핵심**: "어떤 순서"로 일할 것인가?
*   **위치**: `.agent/workflows/`

---

## 💡 구성 요소 간의 관계
이 시스템은 다음과 같이 작동합니다:
1.  **유저**가 **워크플로우** 명령(예: `/plan`)을 내리면,
2.  **워크플로우**에 정의된 절차에 따라 적합한 **역할**이 선택되고,
3.  해당 **역할**은 프로젝트의 **규칙**을 준수하며 필요한 **스킬**을 사용하여 작업을 완수합니다.
