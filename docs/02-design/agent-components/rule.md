# 에이전트 구성 요소: 규칙 (Rules)

규칙(Rule)은 AI 에이전트가 모든 작업을 수행할 때 **반드시 준수해야 하는 강제 지침**입니다. 프로젝트의 일관성, 안전성, 품질을 보장하는 핵심 장치입니다.

## 📝 규칙의 정의
규칙은 "무엇을 해야 하고, 무엇을 하지 말아야 하는지"를 정의합니다. 기술적인 제약 사항부터 코딩 컨벤션, 보안 정책까지 포함됩니다.

## 🛠️ 작성 및 유지 관리 방법
*   **저장 위치**:
    *   `GEMINI.md`: 프로젝트 전체에 적용되는 최상위 규칙.
    *   `.agent/rules/`: 특정 주제별 세부 규칙 파일들.
*   **작성 형식**: 명확하고 단호한 문장으로 작성하며, 위반 시의 대안이나 권장 사항을 함께 명시합니다.
*   **계층 구조**: 프로젝트 전체 규칙(`GEMINI.md`)을 기본으로 하되, 하위 디렉토리나 특정 역할에 최적화된 세부 규칙이 이를 보완합니다.

## 👤 유저가 해야 할 부분
*   **핵심 규칙 설정**: 프로젝트 시작 시 반드시 지켜야 할 기술 스택이나 아키텍처 원칙을 규칙으로 정의합니다.
*   **규칙 업데이트**: 프로젝트가 진행됨에 따라 발견되는 새로운 제약 사항이나 패턴을 규칙에 추가하여 에이전트가 학습하게 합니다.

## 🤖 AI가 자동으로 해주는 부분
*   **실시간 검토**: 코드를 작성하거나 명령을 실행하기 전, 해당 작업이 정의된 규칙에 위반되는지 자동으로 체크합니다.
*   **강제 준수**: 유저의 요청이 규칙과 충돌할 경우, 규칙을 우선시하며 유저에게 이유를 설명하고 대안을 제시합니다.
*   **일관성 유지**: 여러 에이전트가 협업하더라도 동일한 규칙을 공유하므로 일관된 결과물을 만들어냅니다.
